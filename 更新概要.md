# 更新概要

## 版本说明

- **原版**：`index.html` - 基础版本
- **Demo版**：`demo.html` - 可视化调整增强版

## 主要功能差异

### ✨ 新增功能

#### 1. 📊 汇总可视化拖拽调整

**原版**：只能查看各车型的分配比例，无法进行可视化调整。

**Demo版**：新增"📊 汇总"标签，提供交互式柱状图：
- **实时拖拽**：点击并拖动柱状图上下调整每日总量
- **可视化反馈**：柱状图上直接显示数值，无需悬浮查看
- **智能分配**：拖动某天后，其他天自动按比例调整
- **月度守恒**：各车型月度总量保持不变，只调整日分配

#### 2. 🎯 智能比例重分配

**原版**：手动调整需要逐个修改每个车型的每天数据。

**Demo版**：
- 拖动汇总视图中某天的总量
- 系统自动按原始占比分配到各车型
- 未调整天的数量自动重新计算
- 确保每个车型月度目标不变

#### 3. 📈 实时数据反馈

**原版**：静态表格显示。

**Demo版**：
- 拖动时实时更新表格数据（50ms节流）
- 实时更新统计卡片（调整天数、最大调整幅度）
- 实时显示汇总目标量变化
- 所有数值保持整数显示

#### 4. 🎨 视觉增强

**原版**：纯表格展示。

**Demo版**：
- 交互式柱状图，蓝色/橙色区分未调整/已调整
- 柱子顶部直接显示数值（无需悬浮）
- 渐变背景卡片设计
- 调整天数标记（橙色边框）

## 调整逻辑说明

### 原版
- 静态比例分配
- 无法手动调整
- 只能重新生成

### Demo版

#### 拖动调整原理
```
用户拖动第5天从 450 → 600
↓
系统按各车型原始占比分配增加的 150
- W01: 原占比 40% → 分配 60
- W02: 原占比 35% → 分配 52.5 ≈ 53
- X01: 原占比 25% → 分配 37.5 ≈ 37
↓
从其他26天按原始比例"借出" 150
- 第5天变为调整天（固定）
- 其他天按原始比例重新分配剩余量
↓
结果：
✅ W01月度总量 = 1200 (不变)
✅ W02月度总量 = 800 (不变)
✅ X01月度总量 = 600 (不变)
✅ 第5天汇总 = 600 (调整后)
✅ 所有数值为整数
```

#### 精确整数分配算法
```javascript
1. 向下取整所有分配值
2. 计算总差额 = 目标值 - 已分配值
3. 按余数大小降序排列
4. 将差额分配给余数最大的天/车型
5. 确保总和精确等于目标
```

## 数据完整性保证

### 原版
- 基于历史比例分配
- 受目标量约束

### Demo版（新增）
1. **月度守恒**：每个车型月度总量 = 输入的目标量
2. **整数约束**：所有分配值为整数
3. **总和一致**：每日汇总 = 所有车型当日之和
4. **比例保持**：未调整天保持原始比例关系
5. **调整记录**：可随时重置恢复初始状态
