# 商机日节奏预测算法 - 通俗解释

## 一、这个算法要解决什么问题？

**问题**：公司下个月要卖车，需要知道每天大概能卖多少辆，方便安排人力和资源。

**解决方案**：用过去的销售数据来预测未来，就像天气预报一样——看过去几年这个时候的天气，来预测今年会怎样。

---

## 二、核心思想（用饭店订餐来类比）

假设你开了一家饭店，想预测下个月每天会有多少客人来吃饭。

### 🔍 步骤1：收集历史数据

你翻开过去一年的订餐记录本：
- 周一到周五：平均每天 **100人**
- 周末：平均每天 **150人**
- 春节假期：平均每天 **200人**
- 情人节：平均每天 **180人**

### 🧮 步骤2：按日期类型打分

下个月有30天，你给每天打个"预期客流分数"：

| 日期 | 星期 | 类型 | 基础分数 | 说明 |
|------|------|------|----------|------|
| 3月1日 | 周五 | 工作日 | 100分 | 普通工作日 |
| 3月2日 | 周六 | 周末 | 150分 | 周末客流多 |
| 3月3日 | 周日 | 周末 | 150分 | 周末客流多 |
| 3月4日 | 周一 | 工作日 | 100分 | 普通工作日 |
| ... | ... | ... | ... | ... |
| 3月14日 | 周五 | **情人节** | 180分 | 特殊节日 |
| ... | ... | ... | ... | ... |
| 3月29日 | 周六 | 月末周末 | **180分** | 月末冲量 |
| 3月30日 | 周日 | 月末周末 | **180分** | 月末冲量 |

### 📊 步骤3：算出每天的百分比

假设30天总分 = 100×22天 + 150×6天 + 180×2天 = 3,460分

那么：
- 3月1日（100分）占比 = 100 ÷ 3460 = **2.89%**
- 3月2日（150分）占比 = 150 ÷ 3460 = **4.34%**
- 3月14日（180分）占比 = 180 ÷ 3460 = **5.20%**

### 🎯 步骤4：输入目标，拆分到每天

你说："下个月我要接待 **3,000名客人**"

系统就帮你算出：
- 3月1日 = 3000 × 2.89% = **87人**
- 3月2日 = 3000 × 4.34% = **130人**
- 3月14日 = 3000 × 5.20% = **156人**

## 三、实际算法的5大智能点

### 💡 智能1：去年同期优先

**场景**：去年3月8日妇女节，销量特别好，达到 **180辆**。

**算法逻辑**：
- 如果有去年3月8日的数据 → 直接用 **180辆** 作为基础
- 没有的话 → 才看"星期几"的平均值

**为什么？** 因为特定日期（比如双11、618）的销量规律比"周几"更准。

---

### 💡 智能2：过滤异常值

**场景**：你翻开过去52个周一的数据，发现：
- 正常情况：95-105辆
- 有一个周一只卖了 **20辆**（那天停电了）
- 有一个周一卖了 **300辆**（厂家搞活动）

**算法逻辑**：
- 自动去掉最低的10%和最高的10%
- 只用中间80%的数据来算平均值

**结果**：避免极端情况干扰预测。

---

### 💡 智能3：越近的数据权重越高

**场景**：你有过去12个月的周一数据：

| 时间 | 销量 | 权重 |
|------|------|------|
| 12个月前 | 80辆 | 权重 7% |
| 6个月前 | 90辆 | 权重 26% |
| 上个月 | 100辆 | 权重 100% |

**算法逻辑**：计算加权平均值
- 简单平均 = (80+90+100)/3 = 90辆
- 加权平均 = (80×0.07 + 90×0.26 + 100×1.0) / (0.07+0.26+1.0) ≈ **96辆**

**为什么？** 因为最近的数据更能反映当前市场情况（比如新车型上市、竞品变化）。

---

### 💡 智能4：自动学习月末冲量

**场景**：你发现每个月最后3天，销量总是比平时高。

**算法逻辑**：
1. 计算过去6个月的"月末最后3天"平均值 = 120辆/天
2. 计算"月中"（第5-27天）的平均值 = 100辆/天
3. 得出月末倍数 = 120 ÷ 100 = **1.2倍**
4. 预测下个月最后3天时，自动乘以1.2倍

**结果**：
- 普通周五：100辆
- 月末最后一个周五：100 × 1.2 = **120辆**

---

### 💡 智能5：处理特殊情况

#### 场景A：调休上班日
- **问题**：今年10月1日国庆，10月7日（周六）要上班
- **处理**：把10月7日当"工作日"算，不用周末的数据

#### 场景B：促销活动
- **问题**：3月15日要搞促销
- **处理**：在基础分数上 **×1.5倍**（增加50%）

#### 场景C：新车发布
- **问题**：3月20日新车发布会
- **处理**：在基础分数上 **×1.8倍**（增加80%）

## 四、完整案例演示

### 📋 背景
- 车型：理想L9
- 目标：2024年3月预计卖 **3,000辆**
- 3月有31天

### 📊 计算过程

#### 第1步：给每天打分

| 日期 | 类型 | 基础分数 | 特殊情况 | 最终分数 |
|------|------|----------|----------|----------|
| 3/1 周五 | 工作日 | 100 | - | 100 |
| 3/2 周六 | 周末 | 120 | - | 120 |
| 3/8 周五 | 妇女节 | 100 | ×1.5促销 | 150 |
| 3/15 周五 | 工作日 | 100 | - | 100 |
| 3/29 周五 | 工作日 | 100 | 月末×1.2 | 120 |
| 3/30 周六 | 周末 | 120 | 月末×1.2 | 144 |
| 3/31 周日 | 周末 | 120 | 月末×1.2 | 144 |

假设31天总分 = **3,200分**

#### 第2步：算百分比

- 3/1 = 100 ÷ 3200 = 3.125%
- 3/2 = 120 ÷ 3200 = 3.75%
- 3/8 = 150 ÷ 3200 = 4.6875%
- 3/29 = 120 ÷ 3200 = 3.75%
- 3/30 = 144 ÷ 3200 = 4.5%
- 3/31 = 144 ÷ 3200 = 4.5%

#### 第3步：拆分目标

目标3000辆 × 每天百分比 = 每日销量

- 3/1 = 3000 × 3.125% = **94辆**
- 3/2 = 3000 × 3.75% = **113辆**
- 3/8 = 3000 × 4.6875% = **141辆**（妇女节促销）
- 3/29 = 3000 × 3.75% = **113辆**（月末冲量）
- 3/30 = 3000 × 4.5% = **135辆**（月末周末）
- 3/31 = 3000 × 4.5% = **135辆**（月末周末）

所有天加起来正好 = 3000辆 ✓

## 五、Demo版的可视化调整

### 🎨 拖拽调整功能

**场景**：系统预测3月15日卖 **100辆**，但你觉得这天应该多一点。

#### 不调整时：
```
3/15: 100辆
3/16: 95辆
3/17: 98辆
...
月度总量 = 3000辆
```

#### 你拖动柱状图，把3/15调到 **150辆**：

**系统自动做的事**：

1. **增加第15天**：100 → 150（+50辆）
2. **其他天等比例减少50辆**：
   - 3/16从95辆减少：95 × (2950/2900) ≈ **94辆**
   - 3/17从98辆减少：98 × (2950/2900) ≈ **97辆**
   - ...以此类推

3. **确保总量不变**：
   - 调整前：100+95+98+...=3000
   - 调整后：150+94+97+...=3000 ✓

## 八、FAQ（常见问题）

### Q1：如果没有历史数据怎么办？
**A**：系统会用默认规律：工作日权重1.0，周末0.8，节假日0.5

### Q2：为什么不直接平均分？
**A**：因为实际销量有明显的规律：
- 周末比工作日少
- 节假日更少
- 月末有冲量
- 特定日期（618、双11）有促销

### Q3：预测准确吗？
**A**：
- 如果历史数据充足（1年以上） → 准确率通常在85-90%
- 数据越多、规律越稳定 → 越准确
- 但不能预测突发事件（比如疫情、政策变化）

### Q4：调整后总量会变吗？
**A**：
- 原版：总量固定（比例归一化到100%）
- Demo版：
  - 只输入比例：总比例固定100%
  - 输入目标量：各车型月度目标固定，只调整日分配

### Q5：可以预测多车型吗？
**A**：可以！每个车型独立计算，最后汇总。
- L9、L8、L7分别预测
- 汇总视图显示总量
- 拖动汇总后，自动按各车型原始占比分配

## 九、关键公式总结

### 公式1：加权平均（越近权重越高）
```
加权平均 = (数据1×权重1 + 数据2×权重2 + ... + 数据n×权重n) / (权重1 + 权重2 + ... + 权重n)

其中：权重i = 0.95^(n-i)
（最新数据权重=1，越早权重越小）
```

### 公式2：每日占比
```
某天占比 = 该天权重分数 / 全月总分数 × 100%
```

### 公式3：每日销量
```
某天销量 = 月度目标 × 该天占比
```

### 公式4：月末倍数（自动学习）
```
月末倍数 = 过去月末最后3天平均值 / 月中(5-27号)平均值
```

### 公式5：拖拽调整后的重分配
```
调整后其他天的量 = 原始量 × (剩余总量 / 原始其他天总量)

其中：
剩余总量 = 月度目标 - 已调整天的总量
```
